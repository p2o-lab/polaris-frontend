It helps you to control the modules connected to the PFE and perform service orchestration with them. Thus, you can integrate modules by loading its interface descriptions stored in a JSON file. This JSON file can automatically be generated from a MTP of this module.

Modules

The PFE can control multiple modules. Modules are loaded via JSON files which describe the necessary communication variables to interact with.

Service

Each service consists of a finite state machine which is used to control the service. For that, commands are used to trigger state transitions.

Recipes

Recipes are also loaded via JSON. They describe in a declarative way which steps should be executed one after another and which actions should be taken in each step (e.g. starting or stopping a service). Furthermore, each step defines a set of transitions which lead under a defined condition to a next step.

Playlist

You can dynamically enqueue recipes in a playlist which then can started. This allows to define a set of recipes which will be executed one after another.
